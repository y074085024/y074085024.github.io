1. IOC容器

本章包含Spring的IOC容器方面的内容

1.1. Spring IOC容器和Beans简介

本章涵盖Spring Framework实现控制反转的原则（Spring Framework概览）。IOC 也称为依赖关系注入 (DI)，他是一个过程，根据这个过程定义对象的依赖关系（也就是说, 他们使用的其他对象），通过构造参数、工厂方法参数传递实例，在对象构造完成后设置对象的属性或者从工厂方法中返回，然后当bean返回的时候容器负责将依赖关系注入，这个过程从根本上说是反过来的（因此得名, 控制反转），通过使用类或服务定位器模式等机制的直接构造来控制其依赖项的实例化或位置。

org.springframework.beans和org.springframework.context这两个包是Spring IOC的基础。BeanFactory接口提供了高级配置的机制可以管理任何类型的对象。ApplicationContext是BeanFactory的子接口，它增加了如下功能：

  • 集成Spring AOP很容易
  • 消息资源的处理（用于国际化）
  • 事件发布
  • 应用层特殊的上下文，如在web应用中使用的WebApplicationContext

简言之，BeanFactory提供了一个配置框架和基础功能，ApplicationContext提供了更多企业定制的功能。ApplicationContext是BeanFactory的一个完整的超集，仅仅在本章中描述Spring IOC容器时使用。关于更多替代ApplictionContext的BeanFactory的用法，请看The BeanFactory

在Spring中，对象形成了应用程序的骨架，他们被Spring IOC容器进行管理，在容器中叫做beans。bean 是由 spring ioc 容器实例化、组装和以其他方式管理的对象。因此，一个bean是你应用程序中众多对象中的一个，Beans的互相依赖关系反映在容器使用的配置元数据中。

1.2. 容器概览

org.springframework.context.ApplicationContext接口代表了Spring IOC容器，负责实例化、配置和组装beans。容器通过读取配置元数据获取有关要实例化、配置和组装的对象的说明。配置元数据可以是xml、java注解和java代码。它允许您表达组成应用程序的对象以及这些对象之间丰富的相互依存关系。
