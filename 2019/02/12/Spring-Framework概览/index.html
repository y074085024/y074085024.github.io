Spring Framework 概览

Version 5.1.4.RELEASE

Spring使得创建企业级java应用变得很容易。它提供了采用 java 语言的企业环境中所需的一切，支持 groovy 和 kotlin 作为 jvm 上的替代语言，可以根据application的需要灵活地创建多种不同的架构。从 Spring Framework 5.0 开始, spring 需要 jdk 8+ (java se 8+), 并已经为 jdk 9 提供开箱即用的支持。

Spring 支持广泛的应用场景。 在大型企业中, 应用程序通常存在很长时间, 并且必须在 jdk 和应用程序服务器上运行, 这些服务器的升级周期超出了开发人员的控制范围；还有有一些企业可能使用嵌入在服务器中单个的jar文件运行，这个服务器可能运行在云环境中；其他可能是不需要服务器的独立的应用程序（例如批处理或集成工作负载）。

Spring是开源的。它有一个庞大而活跃的社区, 根据各种现实世界的用例提供持续的反馈，这帮助Spring在很长一段时间内成功地进化。

1. 我们所说的“Spring”是什么意思

术语“Spring”在不同的上下文中代表不同的意思。它可以用来指 Spring Framework项目本身, 这就是一切的开始，随着时间的推移，其他的Spring项目已经构建在Spring Framework之上，通常，人们所说“Spring”，表示的是“Spring”的全家桶。本参考文档着重于基础设施：Spring Framework本身。

Spring Framework被分成模块，应用程序可以选择他们所需要的模块，核心容器的核心模块包括配置模块和依赖注入（dependency injection）机制。除此之外，Spring Framework还为不同的应用架构提供基础的支持，包括消息、事务性数据、持久化和web。也包含了基于Servlet的Spring MVC web框架和同时开发的Spring WebFlux响应式web框架。

关于模块的说明：Spring Framework的jar包允许部署到 jdk 9 的模块路径 (‘Jigsaw’)。对于使用Jigsaw的应用程序，Spring Framework 5的jar包附带了 ‘Automatic-Module-Name’ 清单条目, 定义了稳定的语言级模块名称 (“spring.core”, “spring.context”等), 独立于 jar 工件名称 (jar遵循相同的命名模式, 将’.’改为’-‘, 例如”spring-core” and “spring-context”)，当然, Spring Framework的jar包在 jdk8 和9上的类路径上都运行稳定。

2. Spring和Spring Framework的历史

2003年, 作为对早期 j2ee 规范的复杂性的回应, 出现了Spring。虽然有些人认为 java ee 和 spring 是竞争关系，事实上，Spring是对java ee的补充。Spring编程模型不包含java ee全部规范，而是集成了java ee簇中精心挑选的个性化规范：

  • Servlet API (JSR 340)
  • WebSocket API (JSR 356)
  • Concurrency Utilities (JSR 236)
  • JSON Binding API (JSR 367)
  • Bean Validation (JSR 303)
  • JPA (JSR 338)
  • JMS (JSR 914)
  • 以及必要时用于事务协调的JTA / JCA设置

Spring Framework也支持Dependency Injection(JSR 330)和一般注解(JSR 250)规范，应用程序开发人员可以选择这些规范来使用，替代Spring Framework提供的Spring特定机制。

从Spring5开始，Spring需要Java EE 7级别（例如Servlet 3.1，JPA 2.1）作为最低要求，同时在运行时遇到Java EE 8级别的新API（例如Servlet 4.0，JSON绑定API），提供开箱即用的集成。这使得Spring完全兼容Tomcat 8和9，WebSphere 9和JBoss EAP 7。

随着时间的推移，Java EE在应用程序开发中的角色也在不断演变。在Java EE和Spring的早期，创建应用程序以部署到应用程序服务器，而今天，在Spring Boot的帮助下，应用程序以一种灵活易用的方式创建，比如嵌入 servlet 容器和小的的更改。从 Spring Framework 5开始, webflux 应用程序甚至不直接使用 servlet api, 并且可以在不是 servlet 容器的服务器 (如 netty) 上运行。

Spring不断创新和发展，除了Spring Framework之外, 还有其他项目, 如Spring Boot, Spring Security, Spring Data, Spring Cloud, Spring Batch等，请务必记住, 每个项目都有自己的源代码存储库、问题跟踪器和发布节奏。有关 spring 项目的完整列表, 请参阅 spring.io/projects。

3. 设计哲学

当您了解框架时，重要的是不仅要知道它的作用，还要了解它遵循的原则。以下是Spring Framework的指导原则：

  • 提供各个级别的选择。spring 允许您尽可能延迟设计决策。例如, 您可以通过配置切换持久性提供程序, 而无需更改代码。许多其他基础架构相关问题以及与第三方 api 的集成也是如此
