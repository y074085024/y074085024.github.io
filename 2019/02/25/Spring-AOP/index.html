什么是aop

aop，面向切面编程，是用来解决横切性问题的，与业务流程关注的纵向性问题垂直。一般与业务流程的关系不大，其逻辑不会影响到主的业务逻辑、一般aop主要应用有记录日志、权限验证、事务、性能检查等。

spring aop 概念

  • aspect：切面；是一个抽象的概念，是对于所有纵向业务共同特性的一个抽象，抽象成一个抽象的描述体，就像面向对象中对于属性相同事物抽象成一个准确的描述体。xml（spring）和类（aspectj）可以看成是切面的具体实现。
  • join point：连接点；指的是目标对象被选入切入点的方法
  • pointcut：切入点；连接点的集合
  • target：目标对象
  • proxy：代理对象
    weaving：织入；目标对象被增强的过程，或者说是代理逻辑增加到目标对象的过程
  • advice：通知；织入的代码+位置（前置、后置、环绕、异常、最终等）

Spring aop 用法

  1. 开启Aspect语法支持

    Spring aop默认使用jdk动态代理实现织入，如果类实现了接口使用jdk动态代理，如果类没有实现接口，则只能使用cglib代理实现aop。由于spring原来的aop语法晦涩难懂，所以spring支持了AspectJ的aop语法，要spring支持AspectJ语法需要进行配置，xml方式通过配置<aop:aspectj-autoproxy/>，注解方式通过@EnableAspectJAutoProxy方式开启。

  2. 切点表达式

    • execution：匹配方法执行的连接点，这是使用 spring aop 时使用的主要切入点指示符。例子：execution(modifiers-pattern? ret-type-pattern declaring-type-pattern?name-pattern(param-pattern)throws-pattern?)
      修饰符(可空) 返回类型 声明类型（可空） 方法名（参数）异常
    • within
    • this
    • target

  3. introduction

    引入默认实现，可以指定包中的所有类通过一个接口的默认实现，实现指定的接口，即相当于在类中引入了一个接口的默认实现，如果这个类已经手动实现了这个接口，那么就不会引入。
